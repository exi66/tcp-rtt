# tcp-rtt

Приложение для мониторинга задержки (RTT) до игрового сервера на основе перехвата TCP трафика. В отличие от обычного ping (ICMP), измеряет реальное время отклика по тому же пути, по которому идут игровые данные — включая время обработки на стороне сервера.

## Как работает

Приложение перехватывает TCP пакеты на выбранном сетевом интерфейсе. Когда клиент отправляет пакет с данными, фиксируется время отправки и ожидаемый ACK-номер. Когда сервер присылает подтверждение (ACK), вычисляется разница — это и есть RTT.

## Стек

- **.NET 10 / C# / Windows Forms** — UI и основная логика
- **SharpPcap** — перехват сетевых пакетов (требует Npcap)
- **PacketDotNet** — парсинг IP/TCP пакетов
- **ScottPlot** — отображение графика задержки в реальном времени
